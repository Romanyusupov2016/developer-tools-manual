less - вывести содержимое большого текстового файла, с постраничной прокруткой.
clear - очистка экрана.
exit - закрыть терминал.
man command - справка по комманде.
ctrl + a - перейти в начало строки.
ctrl + e - перейти в конец строки.
history n - показать историю комманд, с n-ым количеством последних записей.

date - отобразить текущую дату и время.

jobs - список запущенных фоновых задач.
fg n - перевести фоновую задачу с номером n в foreground(обычный режим).
reboot - перезагрузить компьютер.
halt - выключить компьютер.

stdin 0 - стандартный поток ввода.
stdout 1 - стандартный поток вывода.
stderr 2 - стандартный поток ошибок.
command > file - перенаправление вывода комманды в файл.
command >> file - перенаправление вывода комманды в файл, с добавлением данных в конец файла.
command 2> file - перенаправление вывода потока ошибок комманды в файл.
command 2>> file - перенаправление вывода потока ошибок комманды в файл (с добавлением данных).
command < file - перенаправить комманде ввод данных из файла.
command-1 | command-2 - перенаправить вывод комманды 1 на ввод комманды 2.

top - просмотр запущенных процессов в интерактивном режиме.
ps -aux - список всех запущенных процессов.

chown owner:group file - назначить владельца и группу владельца для файла.

chmod 777 file - назначить права на файл.
chmod -R 777 directory - назначить права на каталог.

passwd - изменить пароль пользователя. 

find - показать все файлы в текущей директории.
find path - показать все файлы в указзаной директории.

find path -iname "grep-шаблон" - регистро независимый поиск файла по имени.

find path -maxdepth 1 -name ".*" - поиск скрытых файлов и папок.

find path -type d -empty - поиск пустых каталогов
grep шаблон file - поиск вхождения подстроки в файле по указанному шаблону.
grep -i шаблон file - регистро независимый поиск вхождения подстроки в файле по указанному шаблону.
whereis - поиск пути исполняемого файла, исходников, man-страниц программы.

curl:
curl URL - выплнить GET запрос.

curl -L URL - выполнить GET запрос, с автмоматическим редиректом.
curl -o file.html URL - скачать файл и сохранить c указанным именем.
curl -O URL - скачать и сохранить файл.
curl --verbose -I URL - выполнить GET запрос, с детальным выводом информации о запросе и ответе.

Архиватор tar:
.tar.bz2 - bzip2
.tar.gz - gzip
.tar.xz - lzma2

-A - добавить tar-файлы в существующий архив.
-c - создать новый архив.
-d - операция сравнения архива с заданной файловой системой.
--diff - проверить различие между архивами.
--delete - удалить файлы из архива.
-r - добавить файлы в существующий архив.
-t - список содержимого архива.
-u - обновить архив.
-x -извлечь файлы из архива.
-C "directory" - сменить директорию перед выполнением операции на directory.
-f file - вывести результат в файл file.
-j - перенаправить вывод в команду bzip2.
-J - перенаправить вывод в команду xz(lzma2).
-p - сохранить все права доступа к файлу.
-v - выводить подробную информацию процесса.
--totals - выводить итогувую информацию завершённого процесса.
-z - перенаправить вывод в команду gzip.

tar --totals -cvf archive.tar file1 file2 file3 - создать архив archive.tar.
tar -tf archive.tar - просмотр содержимого архива.
tar -xvf archive.tar - распаковать архив tar.
tar -xjvf archive.tar.bz2 - распаковать архив bzip2.
tar -xzvf archive.tar.gz - распаковать архив gzip.
tar -xJvf archive.tar.xz - распаковать архив xz(lzma2).
tar -C "test" -xvf archive.tar - распаковать архив в каталог test.
tar -cjvf archive.tar.bz2 file1 file1 file3 - создать и сжать архив с помощью bzip2.
tar -czvf archive.tar.gz file1 file2 file3 - создать и сжать архив с помощью gzip.
tar -cJvf archive.tar.xz file1 file2 file3 - создать и сжать архив с помощью xz(lzma2).

ZIP архивы:
-d - удалить файл из архива.

-9 - наилучшая степень сжатия.

-e - шифровать файлы.

zip -r archive.zip directory - создать zip архив.
unzip archive.zip - распаковать zip архив.

ssh:
ssh server runs on port 22.
ssh login@host - подключение к серверу SSH.
shh login@host -p port_number - подключение к серверу SHH по указанному порту.
ssh-keygen - генерация пары ключей:
    /home/user/.ssh/id_rsa - приватный ключ.
    /home/user/.ssh/id_rsa.pub - публичный ключ.
/home/user/.ssh/authorized_keys - файл хранящий список авторизованных публичных ключей (находится на сервере).
ssh-copy-id login@host - добавление содержимого публичного ключа в файл "authorized_keys" на сервере.
